INTERVIEW NARRATIVE
================================================================================

When asked "Walk me through your GitHub project":

"I built a Threat Intelligence Pipeline that automates what I do in the SOC
every day, but scales it up to what CTI teams need.

It ingests three data sources:
1. Firewall logs (Sophos - my day job)
2. Email headers (phishing detection)
3. SIEM alerts (incident signals)

The system extracts indicators of compromise (IOCs) - IPs, domains, hashes,
emails - then correlates them across all three sources. IOCs appearing in
multiple places = higher confidence = real threat.

I use Claude AI to analyze these patterns, map them to MITRE ATT&CK framework
to understand attacker tactics, and generate structured intelligence reports.

The reports include:
- Threat actor profile and attribution
- Complete timeline of the attack
- All indicators in one place
- Risk score (0-100)
- Recommended IR actions

This shows I understand both sides: SOC operations (logs, alerts) AND threat
intelligence (correlating to find attackers). It's the automation layer
between them.

For a CTI role, I'd extend this to: integrate with threat feeds (VirusTotal,
AlienVault OTX), track threat actors over time, build intelligence dashboards,
and share IOCs with other organizations."

COMMON QUESTIONS
================================================================================

Q: Do I need an API key to start?
A: No! Run threat_intel_pipeline_demo.py first (no API needed). This shows you
   exactly what the system produces. Demo output is in INC-*.json and INC-*.md.

Q: Why Claude and not ChatGPT or other models?
A: Claude is better at:
   - Handling long, unstructured security data
   - Detailed threat analysis
   - Structured JSON output
   - Understanding context across multiple data sources

Q: Can I use my real Sophos logs?
A: Yes! Anonymize them first:
   - Replace real IPs with examples (192.168.x.x, 10.x.x.x)
   - Replace real domains with examples (company-demo.com, evil-c2.ru)
   - Keep the structure and threat patterns
   Then use as input to the pipeline.

Q: How does this help with CTI interviews?
A: It shows:
   1. You understand threat intelligence workflows
   2. You can build production-ready tools
   3. You bridge SOC + CTI (rare combination)
   4. You're serious about the transition (not just talking about it)
   This makes you stand out vs. typical SOC candidates.

Q: Can I modify it for different log formats?
A: Absolutely! The IOC extraction uses regex patterns. Edit:
   - extract_iocs() method for custom patterns
   - MITRE_MAPPING for your threat landscape
   - Chat prompts for different analysis focus
   See README.md for examples.
