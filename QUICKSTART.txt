## 📝 Use Your Own Data

Edit the `main()` function in `threat_intel_pipeline.py`:

```python
def main():
    pipeline = ThreatIntelligencePipeline()
    
    # Replace these with your data
    with open("your_firewall.log") as f:
        firewall_logs = f.read()
    
    with open("your_email_headers.txt") as f:
        email_headers = f.read()
    
    with open("your_siem_alerts.txt") as f:
        siem_alerts = f.read()
    
    # Run pipeline
    report = pipeline.run_pipeline(
        firewall_logs=firewall_logs,
        email_headers=email_headers,
        siem_alerts=siem_alerts
    )
    
    # Save reports
    pipeline.save_report(report, format='json')
    pipeline.save_report(report, format='md')
```

## 🎯 What This Does

**Input:**
```
Raw Security Data
├── Firewall logs (Sophos: blocked connections, IPS alerts)
├── Email headers (SPF/DKIM/DMARC failures, phishing indicators)
└── SIEM alerts (Splunk/ELK: privilege escalation, lateral movement, data exfil)
```

**Processing:**
```
Claude AI Analysis
├── Extracts IOCs (IPs, domains, hashes, emails)
├── Correlates across sources (finds patterns = threat actors)
├── Maps to MITRE ATT&CK framework
├── Scores risk (0-100)
└── Generates intelligence report
```

**Output:**
```
Incident Reports
├── JSON (machine-readable, for tools/dashboards)
└── Markdown (human-readable, for sharing/discussion)
```

## ❓ Troubleshooting

**"ModuleNotFoundError: No module named 'anthropic'"**
```bash
pip install -r requirements.txt
# If that fails:
pip install --break-system-packages -r requirements.txt
```

**"Could not resolve authentication method"**
```bash
export ANTHROPIC_API_KEY="your-api-key-here"
# Then run again
```

**Demo version not generating reports?**
```bash
python threat_intel_pipeline_demo.py
# Check /mnt/user-data/outputs/ for INC-*.json and INC-*.md files
```
